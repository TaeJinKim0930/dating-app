| 위시리스트/코스 생성기 | **나중 단계**(Phase 1~3, 아래 9번). 지금은 미구현, 웹 only |

## 5. 기능 명세
### (a) 초대링크 판별
```
isInvite = URL에 invite / to / from 파라미터 중 하나라도 있으면 true
```
- **true** → 봉투 개봉 연출 실행 (클릭 시 씰 떨어짐 → 덮개 열림 → 편지 상승 → 하트 폭죽 → 카드 reveal + 동선 라인 draw-in)
- **false** → 연출 스킵, 바로 일정. 우측 하단 `💌 초대장으로 보내기` FAB 노출

### (b) URL 파라미터 (개인화)
| 파라미터 | 의미 | 기본값 |
|---|---|---|
| `to` | 받는 사람 | 너 |
| `from` | 보내는 사람 | 나 |
| `date` | 날짜 | 이번 주말 |
| `spot` | 만날 곳 | 부평역 |
| `invite` | 연출 강제 on | (없음) |

**예시 초대링크**
```
https://<id>.github.io/bupyeong-date/?to=지수&from=민준&date=7월19일&spot=부평역3번출구&invite=1
```

### (c) 초대장 링크 생성기
- 그냥 열기 화면의 `💌 초대장으로 보내기` → 받는사람/보내는사람/날짜/만날곳 입력 → 위 형식 링크 **자동 생성 + 클립보드 복사**
- 그 링크를 받은 사람이 열면 → 봉투 개봉 연출

### (d) 지도 (SVG, JS로 생성)
- 부평역 START 깃발 + 번호 핀 + 점선 동선 + 구간별 거리·시간 칩 + 나침반(N) + 축척(100m) + 부평문화의거리 존
- 각 장소 카드: `📍 네이버 지도`(장소) + `🧭 여기까지 길찾기`(직전 지점→여기 네이버 **도보** 길찾기)

### (e) 장난 RSVP
- "데이트할래요?" → `네, 좋아요♡` = 하트 폭죽 + 답장 토스트 / `음...` = 마우스 대면 **도망가는 버튼**

### (f) 접근성/모션
- `prefers-reduced-motion` 이면 연출 스킵, 바로 펼쳐진 상태로 표시

## 6. 코드 구조 (index.html 내부)
- **단일 파일**: `<style>`(CSS) + HTML 골격 + `<script>`(전 로직)
- **데이터 주도**: 상단 `PLACES` 배열이 원본 → `buildMap()`, `buildTimeline()` 가 지도/일정 자동 렌더
- **좌표 투영 공식** (geo → SVG):
  ```js
  REF = 부평역(37.4896, 126.7234)
  ORIGIN = {x:212, y:446}, K = 0.72
  MLAT = 111320, MLNG = 88320   // 위도1도, 경도1도 당 미터(위도 37.49 기준)
  x = ORIGIN.x + (lng - REF.lng) * MLNG * K
  y = ORIGIN.y - (lat - REF.lat) * MLAT * K   // 북쪽이 위(작은 y)
  ```
- 폰트: Google Fonts (`Jua` 제목 / `Gamja Flower` 손글씨 / `Gowun Dodum` 본문) — CDN 로드, 실패 시 시스템 폰트로 폴백
- 외부 의존: 폰트 CDN, 네이버 링크 (둘 다 열람 시 인터넷 필요). 그 외 이미지/스크립트 없음(자체 완결)

## 8. 배포 (GitHub Pages)
**방법 A — 웹 업로드 (CLI·토큰 불필요, 추천)**
1. https://github.com/new → repo명 `bupyeong-date` → Public → Create
2. "uploading an existing file" → `index.html` 드래그 → Commit
3. Settings → Pages → Source: `main` / `(root)` → Save
4. 1~2분 후 `https://<id>.github.io/bupyeong-date/`

**방법 B — CLI (GitHub 인증된 환경에서)**

## 9. 향후 로드맵 (지금 미구현 — 기록용)
목표: "가고 싶은 곳 위시리스트 → 그걸로 데이트 코스 자동 생성"하는 방향. **지금은 웹만, 앱 X.**

| 단계 | 내용 | 기술 | DB 필요? |
|---|---|---|---|
| Phase 1 | 위시리스트 담기·저장·불러오기 + 내 목록으로 코스 생성 | 정적 HTML + **localStorage** | ❌ |
| Phase 2 | 기기 동기화 / 계정 / 공유 편집 | 무료 **BaaS**(Supabase·Firebase 무료티어) | ✅(BaaS) |
| Phase 3 | "주변 자동 추천 / 홍대 코스 자동" | **지도 검색 API**(카카오 로컬 등) or **AI(LLM) API** | ❌(API) |

**결론**: 저장·내 목록 계산까지는 **정적 HTML(localStorage)로 가능**. DB는 "여러 기기 동기화 / 계정 / 공유 편집" 필요해질 때만. 그때도 프론트는 GitHub Pages 유지, 데이터만 무료 BaaS가 담당.

- 확장 대비: 현재 `PLACES` 배열이 데이터 원본이라, 위시리스트 항목도 같은 스키마(`{name, geo, ...}`)로 넣으면 지도·일정 재사용 가능.
